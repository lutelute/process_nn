GPT-2 の生成過程と内部構造

本物の GPT-2 small（124M）をブラウザで forward 実行し、生成を 1 トークンずつ進めながら attention・残差・logit lens・次トークン確率を覗く

← ニューラルネットの学習過程へ

フレームワークを使わず、トークナイザ（byte-level BPE）と forward（埋め込み → Transformer ブロック×12 → ln_f → lm_head）を 素の JavaScript で実装しています。transformers.js は logits しか返さず内部を取り出せないため、 attention や残差を見るには自前 forward が必要、という方針です（詳細は GPT_NOTES.md）。

1. トークナイザ実演 重みDL不要・即動作

GPT-2 の byte-level BPE。テキスト → 事前分割 → バイト列 → BPE マージ → トークン ID。未知語が原理的に存在しません（日本語・絵文字も可）。

vocab.json / merges.txt を読み込み中…

2. 生成過程と内部構造ビューア 要・重み読み込み

本物の重み（model.safetensors, 約 523MB）が必要です。ローカルにあるファイルを選ぶのが最速。 無ければ HuggingFace から取得できます（初回 523MB ダウンロード）。読み込み後、生成を 1 トークンずつ進めて内部を観察できます。

実装は教育目的。素の JS のため WebGPU 等の最適化はしておらず、1 トークンの生成に数百ms〜数秒かかります。トークナイザは Node で encode("Hello world") = [15496, 995] と往復一致を検証済み。forward はコンポーネント単体テストと本物重みでの生成健全性で確認しています。
